# Kieran Klaassen (AI Experience) — investir no sistema, não só no feature
> Cada aprendizado que você extrai e guarda faz a IA render mais na próxima — o valor compõe (compound).

## As sacadas
- **Metade do tempo no feature, metade ensinando o sistema.** *"50% should go into creating the feature... but 50% of the time should go to teaching the system for anything that it did wrong, can we learn something... because it will make the next time better."* `[00:37]`
- **Só dá pra soltar a máquina se você investe no sistema antes.** *"it's very important to be able to let go and let the machine work overnight for many hours in parallel and the only way to be able to do that is making sure you spend time on that system."* `[00:20]`
- **Guardar solução no repositório é mais eficiente em tokens, não menos.** *"it's actually more [context]-efficient because if you have the right answers and the right solutions already within the token, you don't need to review, you don't need to correct, you don't need to do deep research across the internet."* `[01:16]`
- **Funciona porque seu cérebro é fixo e a IA não é.** *"The real reason why this works is my brain [is] fixed and AI isn't or less fixed. And my advice is keep extracting until the complete [system] runs itself and it's so freaking good that it will surprise you."* `[01:37]`
- **Nunca se repita: se você se pegou repetindo, extraia.** *"make sure to extract... If you see yourself repeating yourself make sure to extract it somehow make sure it doesn't happen again."* `[09:32]`
- **O sistema deve rodar sem você — e ser chato de tão previsível.** *"Make sure that there is a [system] that can run without you that does the planning working reviewing and it should be boring. It should just work. You should not be needed."* `[09:37]`
- **Construa um sistema que lembra, em vez de julgar caso a caso.** *"make sure to build a system that will remember instead of, was this good? Make the system better, make the system know."* `[10:59]`
- **A prova de que compôs: o próximo feature fica mais fácil, não mais difícil.** *"your standard should be the next feature should be easier because you shipped this one. If the next feature is harder because you added complexity... we're flipping that."* `[12:11]`

## O que prova pra INEVITA
O ativo não é a resposta de hoje — é o contexto que você guardou e que faz a próxima resposta vir pronta. Investir metade do esforço no sistema (não só no resultado imediato) é o que permite delegar de verdade. Cada átomo guardado é um aprendizado que a IA não precisa redescobrir: o valor compõe, e o próximo movimento fica mais barato porque você fez esse.
